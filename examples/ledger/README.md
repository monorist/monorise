# Ledger Example

A transaction ledgering system built with [Monorise](https://github.com/monorist/monorise) — showcasing date-range queries, multi-dimensional filtering, and custom aggregation endpoints on DynamoDB.

## What it demonstrates

- **Entities**: Merchant, Buyer, Transaction
- **Tags** (date-range queries):
  - All transactions by date range
  - Transactions by merchant + date range
  - Transactions by buyer + date range
  - Transactions by type (sale/refund/discount)
- **Mutuals**: Transaction ↔ Merchant, Transaction ↔ Buyer
- **Custom routes**: Aggregation endpoint that calculates totals for a date range

## API endpoints

After `sst dev`, these endpoints are available:

### Built-in (auto-generated by monorise)

```
POST   /core/entity/merchant                    # Create merchant
POST   /core/entity/buyer                       # Create buyer
POST   /core/entity/transaction                 # Create transaction
GET    /core/tag/transaction/date?start=...&end=...                          # All transactions by date range
GET    /core/tag/transaction/merchant-date?group={merchantId}&start=...&end=... # By merchant + date
GET    /core/tag/transaction/buyer-date?group={buyerId}&start=...&end=...       # By buyer + date
GET    /core/tag/transaction/type?group=sale                                    # By type
GET    /core/mutual/merchant/{merchantId}/transaction                           # Transactions for a merchant
GET    /core/mutual/buyer/{buyerId}/transaction                                 # Transactions for a buyer
```

### Custom

```
GET    /core/app/summary?start=...&end=...      # Aggregated totals (sales, refunds, discounts, net)
```

## Getting started

```bash
cd examples/ledger
npm install
npx sst dev
```

The dashboard will be available at `http://localhost:3000`.
