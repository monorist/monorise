---
name: Always clean stale node_modules after npm install
description: After npm install in the chat example, always rm -rf apps/chat/node_modules/monorise and apps/chat/.next to prevent stale copies shadowing the correct package
type: feedback
---

After every `npm install` of the monorise tgz in `examples/websocket-chat/`, ALWAYS run:
```
rm -rf apps/chat/node_modules/monorise apps/chat/.next
```

**Why:** npm install creates a duplicate copy of monorise inside `apps/chat/node_modules/` that shadows the correct one in the parent `node_modules/`. This causes "X is not a function" errors because the stale copy doesn't have the latest exports.

**How to apply:** Every single time after running `npm install` on the tgz, clean the stale copies. Never skip this step.
